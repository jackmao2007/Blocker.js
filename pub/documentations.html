Documentations

Blocker.js allows users to put restrictions on content and provides a few ways to hide contents, lock contents, show contents, and add effects to unlocked contents. Blocker.js do this by first adding a way to block the content, then set up a way to trigger the blocked content to show, then let the user meet certain conditions, then sets up animations to show the blocked contents, unlock the content, and finally setup after effects for the unlocked contents.

Blocker.js instance

A new blocker instance is created by calling Blocker(query) where query is a CSS selector 
         let blocker = Blocker("#main-body")

Trigger Method

Trigger methods setups up how the reveil process is triggered. This is defaulted to click. 
         blocker.setupTriggerEvent(triggerEvent, triggerEventOptions)
triggerEvent Required triggerEventOptions Type Description
'blockerjsScroll'
scrollTop
Integer
scrollTop is a integer indicating how far from the top must be scrolled to start the reveil. This mode starts the reveil process by scrolling to a certain postion in the window
'blockerjsHold'
holdDuration
Integer
holdDuration is a integer indicating how many miliseconds must the user hold down the mouse on the element to start the reveil. This mode starts the reveil process by holding down the mouse on the element for a set duration
"blockerjsMultiClick"
multiClicks
Integer
multiClicks is a integer indicating how many times must the user click on the element to start the reveil. This mode starts the reveil process by clicking on the element for set amount of times

Action Method

Action method sets up the necessary actions the users must perfrom to reviel the contents. This is defaulted to none 
         blocker.setShowAction(config)

config

config is a object consiting of atributes:
mode
[, params ]
mode params Type Description
"password"
password
String
The password mode has one params password which is the correct password the user must enter. This mode will create a prompt to enter password in the center of the element when triggered. The element is reveiled when the user submit the correct password
"puzzle"
size
color
trueMap
Integer
String
Array[Array[Boolean]]
The puzzle mode has three params. size is the dimensions of the grid generated, color is the fill color of the grid, and trueMap is the grid that the user must match in order to reveil the element. This mode will create a grid in the center of the element when triggered. The user can fill or unfill the grid cells to match the trueMap, once the grid matches, the element is reveiled

Show Method

Show method sets up the how the element is revieled. This is defaulted to none. (This is currently smoothly integrated with .block) 
         blocker.setShowMethod(config)

config

config is a object consiting of atributes:
mode
[, params ]
mode params Type Description
"scratch"
scratchSize
color
percentReveil
Integer
String
Integer
The scratch mode has three params. scratchSize is the size of the scratching brush, color is the color of the scratcher cover, percentReviel is the percentage that needs to be scratched to reveil the entire element. This mode upon reveil will create a scratcher of a set color on the element, the user can scratch off the sratcher to see the content.
"fade"
color
String
The fade mode has three params, the color indicates the color of the block created. This mode upon reveil will create a block of the set color on top of the element and gradually fade away to reveil the contents
"slide"
color
speed
String
Integer
The slide mode has two params. The color indicates the color of the block created, the speed indicates how fast the block slides away. This mode upon reveil will create a block of the set color on top of the element and slide away to the right and eventually disapear at 100% width to reveil the contents.

After Effects

After Effects sets up how the element is treated after the contents are completely reveiled. This is defaulted to none. 
         blocker.setAfterEffect(config)

config

config is a object consiting of atributes:
mode
[, params ]
mode params Type Description
"tremble"
time
trembleMagnitude
Integer
Integer
The tremble mode has two params. time indicates the duration the element will tremble in miliseconds, and trembleMagnitude indicates how big each shakes will be. This mode after content reveil will shake the content for the set time duration.
"recover"
time
Integer
The recover mode has one params, the time indicates how long the element is covered up again in miliseconds. This mode after content reveil will cover up the element again using the original methods.
